# Workflow Git

## Gałęzie (branches)

| Gałąź | Opis |
|-------|------|
| main | Główna gałąź z działającym kodem |
| feature/backend | Rozwój backendu API |
| feature/web-app | Rozwój aplikacji Angular |
| feature/desktop-app | Rozwój aplikacji WinForms |
| feature/mobile-app | Rozwój aplikacji MAUI |

## Konwencja commitów

Każdy commit opisuje co zostało zrobione:

- `feat:` nowa funkcjonalność
- `fix:` naprawa błędu
- `docs:` zmiany w dokumentacji
- `test:` dodanie testów

## Przykładowe commity

```
feat: dodanie logowania użytkownika
feat: lista zadań z filtrowaniem
fix: naprawienie błędu walidacji tytułu
docs: aktualizacja README
test: testy jednostkowe dla TasksController
```
